# OpenCode 懶人包 #07：開工／收工／初始化技能

> 版本：v0.2
> 更新日期：2026-07-17

## 這個懶人包會幫你做什麼？

安裝三個完整的 OpenCode 全域 Skill：

- `startup`：唯讀檢查 AGENTS.md、Git 與 Obsidian 專案狀態
- `shutdown`：敏感資訊與驗證通過後，依專案規則 commit、push、更新駕駛艙
- `project-init`：不覆蓋既有內容地補齊 AGENTS.md、main、GitHub 與選用的 Obsidian 駕駛艙

## 安裝方式

先安裝本章的安裝 Skill：

```bash
npx skills add mathruffian-dot/opencode-lazy-packs --skill opencode-workflow-skills --agent opencode --global --copy --yes
```

接著讓 OpenCode 載入 `opencode-workflow-skills`。它會把隨附的三個完整資料夾複製到：

```text
~/.config/opencode/skills/startup/
~/.config/opencode/skills/shutdown/
~/.config/opencode/skills/project-init/
```

不得以摘要或空白檔取代隨附內容。

## 權限設定

OpenCode 預設可載入 Skill。若使用者有自訂限制，才安全合併以下設定：

```json
{
  "permission": {
    "skill": {
      "*": "ask",
      "startup": "allow",
      "shutdown": "allow",
      "project-init": "allow"
    }
  }
}
```

## 驗證

重新開啟 OpenCode，說「開工」。正確行為應先讀規則與 Git 狀態，不修改檔案。
