# OpenCode 懶人包 #00：環境建置

> 版本：v0.4
> 更新日期：2026-07-17

## 這個懶人包會幫你做什麼？

- 安裝目前的 Node.js LTS
- 安裝 OpenCode CLI
- 安裝 uv
- 設定 OpenCode 模型供應商
- 驗證工具可用

> Git、GitHub CLI 與 GitHub 登入由 **#02：連接 GitHub** 負責。

## 執行原則

先辨識 Windows、macOS 或 Linux。每次安裝前先顯示命令並取得使用者確認；已安裝且版本正常的工具直接跳過。

Windows 原生環境可以使用，但 OpenCode 官方建議需要最佳相容性時使用 WSL。以下命令不可跨 PowerShell、CMD 與 Bash 混用。

## 步驟一：檢查環境

逐一執行，避免某一個失敗後看不到其餘結果：

```bash
node --version
npm --version
opencode --version
uv --version
```

## 步驟二：補裝 Node.js LTS

### Windows PowerShell

```powershell
winget install --id OpenJS.NodeJS.LTS --accept-source-agreements --accept-package-agreements
```

### macOS

```bash
brew install node
```

### Linux

使用發行版套件管理器或 Node.js 官方建議方式安裝目前 LTS，不要把教學固定在過期的大版本。

安裝後重開終端機並確認 `node --version`、`npm --version`。

## 步驟三：安裝 OpenCode

```bash
npm install -g opencode-ai
opencode --version
```

Windows 也可使用 OpenCode 官方文件列出的 Scoop 或 Chocolatey；不要同時混用多種安裝來源。

## 步驟四：安裝 uv

### Windows PowerShell

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### macOS / Linux

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

重開終端機後確認：

```bash
uv --version
```

## 步驟五：設定模型供應商

```bash
opencode auth list
```

若尚未登入任何供應商，請使用者在互動式終端執行：

```bash
opencode auth login
```

登入完成後可用 `opencode models` 確認模型。

## 完成驗證

```bash
node --version
opencode --version
uv --version
opencode auth list
```
